# Trophy System Redesign Plan

## Overview
Redesign achievement trophies to use Pokemon-themed descriptions (like the Sableye "Jewel Hoarder" trophy), reassign Pokemon to better-fit achievements, add tiered progression trophies, and introduce new trophies using the 37 video-enabled assets.

**Key constraint: Each Pokemon appears in exactly ONE trophy.**

## Part 5: Implementation Plan

### Step 1: Update Trophy Definitions (`lib/trophies.ts`)
- Remove old overlapping trophies: `first_deck`, `deck_collector`, `first_export`, `export_veteran`, `limitless_import`, `meta_chaser`, `search_pro`
- Add all new tiered track trophies with proper `id`, `target`, and `video` properties
- Rewrite descriptions for all remaining one-offs
- Ensure every Pokemon appears exactly once

### Step 2: Update Progress Tracking API
- Rename/add action types to match new trophy IDs: `deck_created`, `search_performed`, `export_completed`, `import_completed`, `login_count`
- Implement auto-unlock: when user hits tier 3 (25), automatically unlock tiers 1 and 2 too
- For "total logins" track, increment on every app visit (not streak-based)

### Step 3: UX Enhancements
- Show progress bar on locked trophies in card grid
- In detail modal, show "X / Y" progress for locked trophies
- Add filter tabs for tracks: Deck Builder, Searcher, Exporter, Importer, Logins, One-Offs

### Step 4: Database
- No schema change needed — `user_trophies` table already supports `progress` and `progress_target`
- New trophy IDs will be created on first unlock attempt (INSERT OR IGNORE pattern)

### Step 5: Testing
- Verify all 32 video trophies play correctly on hover
- Test tiered progression (e.g., creating 5 decks unlocks T1 + T2)
- Verify no Pokemon appears in two trophies
